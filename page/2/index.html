矩阵的构造与打印问题的分析与解决

矩阵的构造与打印问题的分析与解决

（一）问题描述

矩阵的构造与打印，具有非常简洁的输入输出：即根据提示在两个程序中输入一个代表矩阵阶数的数字，要求程序输出对应的矩阵。

  • 输入：根据交互语句判定输入矩阵的类型+一个数字（代表矩阵的阶数）
  • 输出：对应的矩阵

两个矩阵分别有图示的数字走向：

（二）基本要求

根据题目的表述以及个人的理解，总结出如下要求：

（1）能够在在程序的开始（也就是在输入矩阵阶数之前）有选择不同矩阵的交互形式，能够实现在螺旋矩阵和蛇形矩阵之间自主选择；

（2）能够实现较大阶数的矩阵的输出（阶数100之内）；

（3）在一次选择并输出矩阵之后，能够实现循环输入，自主控制停止的功能。

（三）系统分析和设计

用二维矩阵来模拟矩阵的概念，必然要考虑矩阵的阶数即容量问题。在主体函数中主要实现两个功能：其一是实现在螺旋矩阵和蛇形矩阵之间自主选择；其二是实现循环输入，能够自主控制停止。另外再定义两个子函数专门解决两个矩阵的核心问题：如何将矩阵阶数这一信息转化为矩阵本身。

（1）螺旋矩阵的解决

1、如何模拟矩阵的概念？

首先肯定是建立二维数组来模拟一个矩阵。

2、以什么单位来思考问题？

根据数字的走向来思考问题。因为螺旋矩阵就是以矩阵的中心按照数的顺序不断地由外部到内部画圈。因此我们以“圈”为单位来考虑问题。

3、确定了以“圈”为单位之后，怎么具体解决问题？

（1）从单个的圈内考虑，矩阵转一圈需要有三次的方向变换。每一次变换方向对应的二维数组的坐标规律都会有很大的变化。因此需要用不同的循环将数字依次填到对应的元素上；

（2）把“圈”当作一个整体。容易想到外部的循环和圈数有关系；

（3）我们发现如果矩阵的阶数是偶数且为n，那么这个矩阵共有n/2圈。如果矩阵的阶数为奇数，依然将圈数认为是n/2圈的话，中间会有一个孤立的数字。因此要事先判断输入的矩阵的阶数的奇偶性加以区分。

4、圈与圈之间的联系

圈数的递增是连接不同圈的关键数字。其中的数字关系是：当i=0（第一圈）就是第一圈的坐标关系；当i=1（第二圈）就是第二圈的坐标关系。

（2）蛇形矩阵的解决

1、如何模拟矩阵的概念？

同样地，用二维数组模拟矩阵。

2、根据什么结构解决问题？

我们观察发现以一个矩阵的中心点（可能是一个数也有可能是一个二阶矩阵的中心），关于中心点对称的蛇形矩阵数，两个数之和为矩阵阶数的平方加一。因此我们只需求出上三角，就可以根据坐标关系和总和的关系求出完整的矩阵。

3、如何求出上三角？

（1）运用和螺旋矩阵一样的思考方式：从数字的走向来思考。

（2）我们发现其实蛇形矩阵的上三角就是由多条对角线组成的。而且数字顺着走向是自然数列。

（3）根据数学规律，就可以根据对角线的条数算出完整的前k条对角线占据的数字个数总和是多少。即k*(k+1)/2.那么显然第k+1条上的注自只需要k*(k+1)/2加上一个数就可以了。

（4）我们发现，虽然for循环是死的，但是坐标是活的。如果前面的完整的对角线的条数k是奇数，那么就是k*(k+1)/2加上横坐标；k如果是偶数就是k*(k+1)/2加上纵坐标。这样用if-else语句就可以控制。

4、统筹

最后结合2、3、就可以先求出上三角，然后求出完整的矩阵。最后输出就可以了。

FOR LOOP 的灵活性？

FOR LOOP 的灵活性？

你貌似在搞很大的事情。

但是你在搞事情之前不能忽视你在写C语言，还是要按照规矩办事。因此你必须了解一下for loop 的基本结构。

For loop 的基础结构

For loop（以下简称for），说结构好像要吓死人，其实不过就两点：控制表达式，执行表达式（就是循环体）。

但是这两点能搞出什么事情来？我没得神通做出不遗漏的讨论，所以就照着书念/滑稽。

还是先看基础结构

我擅自把 for 后圆括号中的控制表达式中的三个语句分别命名为 初值（语句），范围（语句），逼近（语句）。这是我能办到的事。索然现在听起来比较拗口。但是你会发觉它的妙处的。

这三条语句都是描述计数器（“随便”设置的一个变量）的，别把计数器看成是什么不得了的东西。它（计数器）不过就是把循环的各种特性（循环了多少次，怎么把循环的次数和循环体语句挂钩）运用一个变量进行了具象化。

这里姑且把三条语句成为计数器三语句：初值，范围，逼近。

  • 初值：循环的起点，就是循环的初始行动代号
  • 范围：测试条件，就是含有计数器的关系表达式。当这个表达式为假的时候，就跳出循环
  • 逼近：在初值（往往是一个常量）和范围（往往也是一个常量）之间搭起一座跑道，终极目标就是超出范围，终止循环

这里可以运用一个比喻，就是初值是一个各方面还不完美的男孩子，但是他喜欢一个他心目中理想的女孩子，女孩子心中理想的标准的男孩子是范围（注意到了，初值和范围都是一个点的概念）。然后初值为了达到范围的标准，于是进行了一个计划，就是逼近（这是一个运动的概念）。最后的结果就是，初值通过逼近，最终超出了范围，成功脱团/笑（就是跳出循环）。

这样就差不多理解 for loop 的基本结构了。但是电脑屏幕上得来终觉浅，觉知此事要敲代码并编译运行。下面看一段非常简单（到弱智）的代码。

灵活性也有高低之分

别看灵活性这个词看起来比较高级，但是其实有些灵活性真的是太容易想出来了，就很尴尬，配不上它高大上的称呼啊。

灵活性分为：基本灵活性、进阶灵活性。

基本灵活性

1、改递加为递减，甚至做出不可描述的事情

好的，这非常简单。你看上面这个代码：

for (i = 1; i < n; i++)

其实呢，这个加加可以改为减减。然后还能更加的过分，比如自乘，自除，甚至改为递增2或3或4或任意一个数字。基本除了自刎什么都可以做的出来，关键是要符合基本语法和逻辑。别的就没有什么好说的了。

2、把数字改成字符，看似高端其实也没啥

因为字符在计算机中的存储就是一个数字，只要不要再定义类型，输入输出的时候把类型问题搞错就可以了。

3、测试条件其实可以很”目标化“

这个被我们调侃为”理想男孩标准“的范围就是测试条件，它不一定就是计数器的一个值得限定，还有可能是限定一个关于计数器的表达式的值。就是这样，这一点由于没有限制别的变量参与，如果循环体中不断变化的变量参与进来的话……先不要想不要想。

6.1.1 程序注释

对于这一个循环语句，只有循环体最后一句的东西需要讨论。

根据我们对这一语句的分析，这一语句是判断语句，其作用是输入一个数字的时候转变为1，输入一个字母的时候转变为不是1 的数字。

书的构思：

目的：

  1. 每次循环都要获得一个num 的值（输入字母结束）；
  2. 每次输入数字的时候都要重置status；

实现方法：

利用scanf() 的两个特性来完成。

  1. scanf() 就是用来读取数值的大小的嘛；
  2. scanf() 返回成功读取项的数量—简单理解就是scanf()语句的值为读取成功的项的数量（格式正确就读取了）；

这样的话就成功地利用一条语句更新了两个变量的值—> num, status.

这样就导致了这样一个结果：对于数字的变量成功存储到num 中，不是数字的变量不能成功存储，于是scanf() 语句的值就为0，这样当用户结束的时候，引导用户输入不是数字的q就行了。

注：我不懂的地方：这里作者提到了一个特殊情况：如果scanf() 在转换值之前出了问题（检测到文件结尾或遇到硬件问题），会返回一个特殊值EOF（其值通常被定义为-1），这个值也会使循环终止。—为什么会这样呢？我不晓得。

比较其他解决方案

假如scanf() 没有返回值，那么：

1.用num 的值来判断是否终止循环—>会导致有一些值可能不能参与求和，直接就结束了；

2.在每一次循环之后放置一个判断语句，用户通过输入来决定是否再次执行循环体或者输出求和结果—>减慢了输入速度，不流畅；

总而言之，不如利用scanf() 的返回值来得顺畅。

6.2 While 语句

while 循环语句的通用形式：

statement 部分可以是一句语句，也可以是多句语句；区别是，一条语句不用花括号，多条语句要用花括号。

expression—任何关系表达式—值之间的比较

注：每一次循环都称为一次迭代（iteration）。

6.2.3 while：入口条件循环

while 循环是使用入口条件的有条件循环。所谓“有条件”值得就是语句部分的执行取决于测试表达式描述的条件，如（index < 5)。该表达式是一个入口条件（entry condition），因为必须满足条件才能进入循环体。在有的情况中，就不会进入循环体，因为为条件一开始就为假。

6.13 关键概念

现在是愉快抄书环节

  • 创建循环：

  3. 确保每次测试的值要更新（和第一点遥相呼应）；
  • 使用函数的三步：
  1. 通过函数原型声明函数；
  2. 在程序中调用函数；
  3. 定义函数；

现代的编程习惯是吧程序要素分为结口部分和实现部分，更有利于程序员的思考，和程序框架的搭建。
